[0.4.1]  2026-04-25
---------------------
Neue Features:

  Nennbuero
  + Startnummern pro Klasse: jede Klasse zaehlt ab 1, gleiche Nummer
    kann in verschiedenen Klassen vergeben werden
    (UNIQUE-Constraint von event_id auf class_id umgestellt)
  + "Auto ab 1"-Button pro Klasse: vergibt 1, 2, 3 ... alphabetisch
    nach Nachname; fragt bei vorhandenen Nummern nach Bestaetigung

  Nachrichten
  + Neue Seite /nachrichten fuer alle angemeldeten Mitarbeiter
    (ausser viewer): freie Textnachricht an alle verbundenen Clients
  + 5 Schnell-Buttons (Bratwurst, JL-Besprechung, Pause,
    Siegerehrung, Ruhe im Fahrerlager)
  + Absender-Name im cyan Banner sichtbar (z.B. "Nachricht (admin)")
  + Sitzungs-Log der gesendeten Nachrichten auf der Seite

Bugfixes & Technische Aenderungen:
  ~ conftest.py: class_id-Fixture erstellt Klasse mit run_status
    "running" statt "planned" -> 9 CI-Test-Failures behoben
    (_require_class_running blockierte Ergebnis-Erfassung in Tests)
  ~ participants.py: toter erster Query in _suggest_class entfernt
    (verwendete birth_year als Spaltenname statt als Parameter ->
    sqlite3.OperationalError in 2 Tests)
  ~ schemas.py: RunPenaltyCreate.result_id auf Optional[int] = None
    gesetzt (Path-Parameter kommt aus URL, nicht aus Request-Body ->
    422 Unprocessable Entity in 2 Tests)
  ~ database.py: Migration _migrate_participants_unique_per_class()
    fuer bestehende Datenbanken (UNIQUE-Constraint-Aenderung)

[0.3.0]  2026-04-24

---------------------

Neue Features:



  Zeitnahme

  + Komma als Dezimaltrennzeichen akzeptiert (z.B. 56,13 = 56.13 s)

  + Queue springt nach Bestaetigung automatisch auf den naechsten Starter

    (Bugfix: participant_id fehlte im v_run_results-View)



  Schiedsrichter

  + Lauf-Spalte in der Ergebnistabelle (Training grau, Lauf N blau)

  + Strafen-Korrektur fuer Wertungslaeufe: bestehende Strafen aendern und

    neue hinzufuegen; Live-Vorschau Rohzeit + Strafzeit = Gesamtzeit

  + Training: nur Rohzeit und Status korrigierbar (keine Straffelder)



  Livetiming

  + Lauf-Detailzeilen pro Fahrer: Rohzeit + Strafzeit (z.B. Lauf 1  56.13 +12.0s = 68.13)

  + Strafzeit immer angezeigt, auch wenn 0.0s (grau)



  Raspberry Pi

  + Neuer Client fuer MAX7219 LED-Matrix-Display (SPI, luma.led_matrix)

    als Alternative zum TM1637-Client



Technische Aenderungen & Bugfixes:

  ~ DB-Schema: v_run_results + participant_id-Spalte; alle Views nun mit

    DROP VIEW IF EXISTS + CREATE VIEW (statt CREATE VIEW IF NOT EXISTS)

    damit Schema-Aenderungen beim Neustart immer wirksam werden

  ~ Pydantic-Schemas: path-param-Felder (event_id, reglement_id etc.)

    in Create-Schemas auf Optional[int] = None gesetzt (Validierung

    laeuft vor dem Router-Setzen der Werte)

  ~ participants-Router: Decorator "/" -> "" (kein Trailing-Slash mehr);

    POST-Requests liefen sonst auf 405 Method Not Allowed

  ~ Test-Fixtures: Benutzernamen t_admin/t_schiri/t_zeit/t_nenn statt

    admin/schiri/zeit/nenn (Konflikt mit schema.sql-Seed-User behoben)

  ~ Zeitnahme: doppelter Enter-Handler entfernt (@keydown.enter +

    window-keydown liefen gleichzeitig -> 409 "bereits vorhanden")



[0.2.0]  2026-04-24

---------------------

Neue Features:



  Lichtschranken-Integration (Hauptfeature)

  + Backend-Endpoint /ws/timing fuer externe Zeitmessgeraete (WebSocket)

  + Raspi-Client (tools/racecontrol_client.py): GPIO-Lichtschranke an

    Raspberry Pi; TM1637-Display lokal + Auto-Reconnect per WLAN

  + ELV LSU200 Client (tools/lsu200_client.py): USB-Anbindung der

    ELV LSU200 Lichtschranke (COM-Port, 19200 Baud, Auto-Detect)

  + ZeitnahmeView: automatische Zeituebertragung aus Lichtschranke ins

    Eingabefeld (gruen blinkend); Strafen und Bestaetigung wie gewohnt

  + Lichtschranken-Statusindikator (gruen/grau) in der Zeitnahme

  + Heartbeat-Mechanismus: Backend trackt verbundene Messgeraete



  Druck & Export

  + Ergebnisliste drucken: A4 quer, ADAC Hessen-Thueringen Standardformat

    (Fahrzeit / Fehlerpunkte / Gesamt pro Lauf, Jahrgang, Summe, Differenz,

    Einspruchfrist-Zeitstempel, Schiedsrichter-Unterschriftszeile)

  + Sprecherliste drucken: A4 hoch, alle Teilnehmer nach Klasse mit

    Startnummer, Name, Verein, Jahrgang, Notiz-Spalte

  + Offizielles ADAC-Nennungsformular: 2-seitiger Ausdruck pro Teilnehmer

    (Nennungsformular + Haftungsausschluss); verfuegbar ab Lizenz + Helm + Geld

  + Urkunden: Clubmeisterschaft-Vorlage (Gestaltung mit Goldrahmen) und

    ADAC-Vordruck (Overlay-Positionierung auf vorgedrucktem Formular)



  Echtzeit & PWA

  + WebSocket Live-Push: Zeitnahme, Livetiming und Landingpage reagieren

    sofort auf neue Ergebnisse und Statuswechsel

  + PWA-Unterstuetzung: Service Worker (Workbox), Offline-Caching fuer

    Standings und Public-API, Web App Manifest, "Zum Startbildschirm"

  + Offline-Indikator: Amber-Banner bei Netzwerkausfall (alle Views)

  + Veranstaltungstag-Deployment: npm run build einmalig, FastAPI liefert

    gebauetes Frontend direkt aus; kein Node.js am Veranstaltungstag



  Admin

  + Test-Tab im Admin-Bereich: API-Verbindungstest (JWT-Check) und

    Testdaten-Seeder (Reglement, Klassen, Teilnehmer) mit Live-Fortschrittslog

  + GPL-2.0 Lizenzseite (/lizenz) mit Projektinfo, Bibliotheksuebersicht

    und Link auf gnu.org; Lizenz-Link in der Statusbar



  Qualitaetssicherung & CI/CD

  + pytest-Integrationstests: auth, events, participants, results, public

    (40+ Tests, In-Memory-SQLite, JWT-Fixtures, Role-Guard-Checks)

  + Vitest-Tests: useNetworkStatus und useRealtimeUpdate (12 Tests)

  + GitHub Actions CI/CD: backend-tests (Python 3.11) + frontend-tests

    (Node 20) + frontend-build; automatisch bei Push/PR auf main



Technische Aenderungen:

  ~ API-Routing: einheitliches /api-Praefix fuer alle Endpunkte

    (Dev-Proxy und Produktion konsistent)

  ~ ZeitnahmeView nutzt jetzt useRealtimeUpdate-Composable fuer WS

  ~ StatusBar: GPL-2.0 Link neben Versionsanzeige

  + Route /lizenz (public, kein Login erforderlich)

  + tools/ Verzeichnis: externe Geraete-Clients und deren requirements.txt





[0.1.0]  2026-04-22

---------------------

Erste vollstaendige Version fuer den Einsatz beim MSC Braach e.V. im ADAC.



Neu:

  + Oeffentliche Landingpage (/) mit Veranstaltungsinfos, Klassenstatus,

    Zwischenwertung, Sponsoren und Beschreibungstext

  + Online-Selbstanmeldung (/nennen) fuer Teilnehmer vor Ort (kein Login)

    mit Duplikatpruefung nach Lizenznummer und Name/Jahrgang

  + Livetiming (/livetiming) ohne Login

  + Nennbuero: Teilnehmerverwaltung, Abnahme (Nenngeld + Helm),

    Statusworkflow (gemeldet -> eingecheckt -> freigegeben)

  + Startnummernvergabe-Modus nach der Auslosung

  + Nennungsschluss pro Klasse mit Startzeiterfassung

  + Nennliste drucken mit Versicherungshinweis und Elternunterschriftfeld

  + Zeitnahme: Zeiteingabe, Strafbuchung per Button/Tastaturkuerzel,

    DNS/DNF/DSQ, Undo-Funktion, Keyboard-Shortcuts

  + Vorziehen-Funktion in der Zeitnahme: beliebigen Fahrer an die

    Startposition 1 setzen (manuelle Warteschlangen-Umstellung)

  + Schiedsrichterview: Klasse starten, unterbrechen, fortsetzen,

    beenden; 30-Minuten Einspruchfrist-Timer; Ergebniskorrekturen

    mit Pflichtbegruendung und Audit-Log

  + Klassenstatus: geplant / laeuft / unterbrochen / vorlaeufig / offiziell

  + Admin-Bereich: Veranstaltungen (CRUD inkl. Loeschen), Klassen,

    Vereine, Benutzer, Sponsoren, Systemeinstellungen

  + Sponsorenverwaltung mit Logo-URL, Website-Link, Sortierung,

    Aktiv/Inaktiv; erscheinen auf der Landingpage

  + Veranstaltungsbeschreibung (Infotext auf der Landingpage)

  + Systemeinstellungen: Druckvorlage (Veranstalter, Adresse,

    Versicherungshinweis, Einverstaendniserklaerung)

  + Reglementverwaltung: Scoring-Typen, Straf-Definitionen

  + Mannschaftswertung (Teams, bis zu 4 Mitglieder, beste 3 gewertet)

  + Rollenbasierte Zuganskontrolle: admin, nennung, zeitnahme,

    schiedsrichter, viewer

  + JWT-Authentifizierung (HS256), bcrypt-Passwoerter

  + SQLite mit WAL-Modus fuer parallele Zugriffe

  + Automatische Datenbank-Migration beim Start (verlustfrei)

  + Logout leitet auf die oeffentliche Landingpage weiter



Technische Basis:

  Backend:   Python 3.9+, FastAPI, SQLite, PyJWT, bcrypt

  Frontend:  Vue 3 (Composition API), Vite, Pinia, Vue Router,

             Tailwind CSS, Axios

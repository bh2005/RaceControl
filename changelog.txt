    in der Wertung mit korrektem Rang, erhalten aber points=0 statt
    KS2000-Punkte — entspricht ADAC-Regelwerk (nur Lizenznehmer erhalten Punkte)

  ESP32 Bluetooth SPP Lichtschranke (neu – Variante N)
  + Neues PlatformIO-Projekt: lichtschranken/ESP32/esp32_bluetooth_lichtschranke/
  + Kein WLAN/Router nötig: ESP32 verbindet sich direkt per Classic Bluetooth
    (SPP) mit dem Laptop — erscheint als /dev/rfcomm0 (Linux) oder COM-Port
    (Windows); serial_logger.py läuft ohne Code-Änderung
  + DIP-Schalter: gleiche Belegung wie Variante L (SW1=Spur, SW2+SW3=CP)
  + BT-Name automatisch aus DIP-Schaltern: "RC-BT-A-0", "RC-BT-B-3" etc.
  + ISR mit Debouncing, LED-Heartbeat (langsam=kein Client, kurz=Trigger)
  + JSON-Format identisch mit WiFi/LoRa-Sender (type/unix/lane/cp/ms)
  + USB-Serial parallel aktiv (Debug / Fallback bei Kabelanschluss)
  + Szenario-Dokumentation: lichtschranken/ESP32/SZENARIO_BLUETOOTH.md
    (Laptop-Setup rfcomm, systemd-Unit, Mehrgerätebetrieb, Entscheidungshilfe)

  Netzwerk-Infrastruktur Scripts
  + server/set_infra_openwrt.sh: DNS-Eintrag jetzt per "uci add_list
    dhcp.@dnsmasq[0].address=" statt konfdir-Datei — funktioniert auf
    allen OpenWrt-Versionen zuverlässig, überlebt Neustart (uci commit)
  + server/set_infra_laptop.sh: iptables Port 80→1980 per PREROUTING,
    persistiert via iptables-persistent oder systemd-Unit
  + http://racecontrol erreichbar für alle WLAN/LAN-Geräte nach Script-Ausführung

[0.9.1] - 2026-05-07
---------------------
Lichtschranken-Erweiterung: Arduino, ESP32, LoRa, GPS + Ordnerreorganisation

  Hardware -- Arduino Nano Sketche (PlatformIO, lichtschranken/Ardoino/)
  + arduino_TM1637_lichtschranke: Standalone-Zeitmessung, 4-Digit-Display,
    USB-Serial JSON-Ausgabe bei jedem Ergebnis
  + arduino_MAX7219_lichtschranke: wie TM1637, aber 8-Digit-Display mit
    Live-Anzeige waehrend der Fahrt (50 ms Refresh)
  + arduino_lora_sender: LoRa 433 MHz Sender (SX1276/RA-02) mit DS3231 RTC;
    UTC-Timestamp beim Interrupt gestempelt; Zeitsync-Protokoll vom Gateway;
    Fallback unix=0 wenn kein RTC

  Hardware -- ESP32 Sketche (PlatformIO, lichtschranken/ESP32/)
  + esp32_tm1637_lichtschranke: TM1637 + DIP-Schalter (SW1: Standalone/WiFi);
    sendet Ergebnis per HTTP an RaceControl wenn WiFi aktiv
  + esp32_max7219_lichtschranke: MAX7219 + DIP-Schalter (SW2: Helligkeit);
    Live-Anzeige (30 ms), WiFi-Modus wie TM1637-Variante
  + esp32_lora_wifi_sender: WiFi primaer (NTP-Sync, HTTP-POST direkt) +
    LoRa automatischer Fallback; DS3231 RTC; DIP-Schalter fuer Spur (A/B)
    und Checkpoint (0=start, 1-2=split, 3=finish)

  Hardware -- Raspberry Pi LoRa-Gateway (lichtschranken/RasPi/)
  + lora_ziel_gateway.py: empfaengt start/split/finish-Pakete von
    Arduino/ESP32-Sendern per LoRa; leitet per HTTP an Backend weiter;
    sendet alle 60 s Zeitsync-Broadcast; Fallback: Gateway-Timestamp
    wenn unix=0 im Paket

  Tools -- GPS-Timesync (lichtschranken/GPS/)
  + ublox8_gps_client.py: Live-Monitor erweitert um WebSocket-Anbindung;
    sendet alle 10 s GPS-UTC-Zeit als gps_timesync ans Backend wenn
    Fix vorhanden und >= 4 Satelliten; Intervall per .env konfigurierbar

  Tools -- Serial Logger
  + serial_logger.py: liest JSON-Zeilen von Arduino/ESP32 ueber USB-Serial;
    farbige Terminal-Ausgabe; CSV-Log (zeitmessung_DATUM.csv);
    auto-detect COM-Port; unterstuetzt 9600 (Nano) und 115200 (ESP32) Baud

  Tools -- Bugfixes
  + alge_timy_client.py: TIMING_API_KEY fehlte; wird jetzt als ?key=...
    an BACKEND_WS angehaengt (wie lsu200_client.py)
  + alge_multi_timy_client.py: TIMING_API_KEY aus .env; ?key=... an WS-URL

  Infrastruktur

    Ardoino/, ESP32/, RasPi/, GPS/, ALGE/, LSU200/
  + Arduino_lichtschranke/ und RaPi_lichtschranke/ und tools/ geloescht
  + setup_tools.sh: neues Einrichtungs-Script fuer alle Clients
    (Python-Version pruefen, pip install, .env anlegen, API-Key auto-fetch)
  + .env.example: TIMY_START_PORT / TIMY_FINISH_PORT ergaenzt
  + platformio.ini in allen 6 Sketch-Ordnern (Nano + ESP32)

  Frontend
